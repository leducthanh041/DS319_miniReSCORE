# 01. Session Summary

## 1. Mục tiêu của các chỉnh sửa gần đây

- Chuyển các script build / download sang chế độ ghi log ra file để dễ debug.
- Sửa lỗi build retrieval DB cho HotpotQA.
- Rà soát lại link download dataset trên web và sửa các URL sai / không ổn định.
- Giảm lỗi OOM khi embed passages bằng cách ép dùng GPU trống.
- Cho phép `build.sh` resume từ bước đã hoàn tất thay vì chạy lại từ đầu.

## 2. Các lỗi đã xác định và cách xử lý

### 2.1. `build.sh` trước đó chỉ in lỗi ra terminal

- Đã sửa `script/download/build.sh` để mỗi bước ghi log vào:
  - `logs/build/<timestamp>/build_summary.log`
  - `logs/build/<timestamp>/<dataset>__<step>.log`
- Có hỗ trợ `BUILD_LOG_MIRROR=1` nếu muốn vừa ghi log vừa hiện full output trên terminal.

### 2.2. HotpotQA preprocess bị rỗng dẫn đến crash ở bước embed

Nguyên nhân:
- Raw corpus được tải về dưới thư mục:
  - `data/raw_data/hotpotqa/wikpedia-paragraphs`
- Nhưng code preprocess lại tìm:
  - `data/raw_data/hotpotqa/wikipedia-paragraphs`

Hậu quả:
- `data/embed_ready_data/hotpotqa.tsv` chỉ có header.
- `generate_passage_embeddings` crash vì không có passages để `torch.cat()`.

Đã sửa:
- `source/run/preprocess_raw_data.py`
  - chấp nhận cả `wikipedia-paragraphs` và `wikpedia-paragraphs`
  - báo lỗi rõ nếu không tìm thấy corpus hoặc sinh ra 0 passage
- `source/run/generate_passage_embeddings.py`
  - fail sớm với thông báo rõ nếu input passages rỗng
  - dùng đúng các CLI args thay vì hardcode
- `script/download/multihop_raw_data.sh`
  - chuẩn hóa tên thư mục HotpotQA corpus về `wikipedia-paragraphs` cho các lần download sau

### 2.3. Một số link download dataset bị sai hoặc không ổn định

Đã rà soát lại trên web và sửa:
- HotpotQA:
  - đổi từ `http://curtis.ml.cmu.edu/data/hotpot/...`
  - sang `http://curtis.ml.cmu.edu/datasets/hotpot/...`
- MuSiQue:
  - dùng `gdown --id 1tGdADlNjWFaHLeZZGShh2IRcpO6Lv24h`
- 2WikiMultiHopQA:
  - giữ flow tải `data_ids.zip`
- IIRC:
  - thêm fallback giữa:
    - `https://iirc-dataset.s3.us-west-2.amazonaws.com/...`
    - `https://iirc-data.s3.us-west-2.amazonaws.com/...`

Ngoài ra `script/download/multihop_raw_data.sh` đã được sửa để:
- ghi log ra `logs/download/raw_data/<timestamp>/`
- retry download
- skip các phần đã có đủ file
- xóa file temp cũ trước khi tải lại để tránh reuse archive hỏng / thiếu

### 2.4. OOM ở bước `generate_passage_embeddings`

Log cho thấy tiến trình đang cố load model lên GPU vật lý `0`, trong khi GPU này gần đầy.

Đã sửa `script/download/build.sh`:
- mặc định dùng `BUILD_CUDA_VISIBLE_DEVICES=2`
- giảm `--per_gpu_batch_size` mặc định từ `1024` xuống `128`
- thêm `PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True`

Ví dụ các lệnh nên dùng:

```bash
bash script/download/build.sh
```

```bash
BUILD_CUDA_VISIBLE_DEVICES=3 bash script/download/build.sh
```

```bash
BUILD_CUDA_VISIBLE_DEVICES=2 BUILD_EMBED_BATCH_SIZE=64 bash script/download/build.sh
```

### 2.5. `build.sh` chưa thể tiếp tục từ bước dở dang

Đã sửa `script/download/build.sh` để tự skip bước đã hoàn tất:
- skip `preprocess_raw_data` nếu TSV đã tồn tại và có dữ liệu
- skip `generate_passage_embeddings` nếu đã có `embeddings_*`
- skip `build_index` nếu đã có:
  - `index.faiss`
  - `faiss_id_to_docstore_id.pkl`
  - `docstore.db`

Nếu muốn ép chạy lại từ đầu:

```bash
BUILD_FORCE_REBUILD=1 bash script/download/build.sh
```
