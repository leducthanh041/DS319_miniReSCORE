# 10. Training & Inference vLLM Summary

## Scope

Các thay đổi này tập trung vào việc chạy `train.py` và `inference.py` với Llama 3.1 8B, vLLM, GPU RTX 2080 Ti, log runtime, và hạn chế lỗi CUDA/OOM.

## Inference

- `source/run/inference.py` đã được chỉnh để ghi runtime log ra `./logs/inference/<dataset>/<running_name>__<timestamp>/inference.log`.
- Thêm log cấu hình quan trọng khi chạy inference: dataset, split, data file, database path, prompt set, CUDA devices, generator config, retriever config, output paths.
- Sửa import retrieval step từ `source.pipeline.step.__retrieval` sang `source.pipeline.step.retrieval`.
- Thêm `VLLM_WORKER_MULTIPROC_METHOD=spawn` và `multiprocessing.set_start_method("spawn")` để tránh lỗi CUDA fork với vLLM.
- Giữ vLLM là mặc định cho inference, có thêm `--disable_vllm` chỉ làm fallback debug.
- Thêm `--generation_dtype`, mặc định `half`, để RTX 2080 Ti chạy được vì GPU này không hỗ trợ `bfloat16`.
- Thêm `--generation_tensor_parallel_size` để kiểm soát số GPU vLLM sử dụng.
- Thêm `--generation_max_model_len` để tách KV-cache length của vLLM khỏi logic `generation_max_total_tokens`, giúp tránh lỗi `# GPU blocks: 0`.
- Thêm `--generation_gpu_memory_utilization`, `--generation_swap_space`, `--generation_cpu_offload_gb`; mặc định hiện tại giữ GPU-only với `swap_space=0` và `cpu_offload_gb=0`.
- Thêm `--database_path` để override retrieval DB khi cần.
- Thêm `--retriever_device` để chủ động đặt retriever lên GPU, ví dụ `cuda:0`.

## Training

- `source/run/train.py` đã thêm cấu hình vLLM dạng opt-in bằng `--generation_use_vllm`.
- Mặc định training vẫn không bật vLLM để tránh thay đổi behavior/loss path cũ ngoài ý muốn.
- Khi bật `--generation_use_vllm`, scorer dùng `prompt_logprobs` từ vLLM thay cho forward logits của Hugging Face.
- Thêm các flag tương ứng với inference: `--generation_dtype`, `--generation_tensor_parallel_size`, `--generation_gpu_memory_utilization`, `--generation_max_model_len`, `--generation_swap_space`, `--generation_cpu_offload_gb`.
- Giữ runtime log, early stopping, validation limit, checkpoint best validation và safe CUDA cache handling từ các lần sửa trước.

## Llama Generator

- `source/module/generate/llama.py` đã thêm `dtype`, `tensor_parallel_size`, `max_model_len`, `swap_space`, `cpu_offload_gb` vào `LlamaGeneratorConfig`.
- vLLM hiện khởi tạo với `dtype="half"` mặc định để phù hợp RTX 2080 Ti.
- vLLM hiện dùng `max_model_len` riêng để giảm KV-cache footprint khi cần.
- Bỏ `length_penalty` khỏi `SamplingParams` của vLLM vì vLLM `0.6.3.post1` trong env hiện tại không hỗ trợ tham số này.
- Thêm `_score_with_vllm()` dùng `prompt_logprobs` để training có thể tính perplexity-like score khi bật vLLM.
- Giữ attention mask và cleanup generation args cho nhánh Hugging Face từ các lần sửa trước.

## Current GPU-Only Inference Recommendation

Với 2 GPU RTX 2080 Ti 11GB, nên chạy GPU-only bằng cách giảm KV cache:

```bash
CUDA_VISIBLE_DEVICES=5,6 python -m source.run.inference \
  --method rescore \
  --running_name infer_musique_step690_vllm_gpu_only \
  --dataset musique \
  --dataset_split test \
  --prompt_set 1 \
  --retrieval_query_model_name_or_path ./predictions/musique/train_musique_resume_best___llama_3.1_8b_instruct___best_validation/multi_retrieval___train/prompt_set__1/retr_count__4/epoch_0_step_690 \
  --retrieval_passage_model_name_or_path facebook/contriever-msmarco \
  --retrieval_count 4 \
  --retrieval_buffer_size 8 \
  --retrieval_batch_size 8 \
  --retrieval_use_fp16 \
  --retriever_device cuda:0 \
  --generation_dtype half \
  --generation_tensor_parallel_size 2 \
  --generation_gpu_memory_utilization 0.95 \
  --generation_max_model_len 2048 \
  --generation_max_total_tokens 2048 \
  --generation_max_batch_size 1 \
  --generation_max_new_tokens 64 \
  --generation_min_new_tokens 1 \
  --generation_swap_space 0 \
  --generation_cpu_offload_gb 0 \
  --max_num_thought 3
```

Nếu vẫn gặp `No available memory for the cache blocks`, giảm tiếp `--generation_max_model_len` và `--generation_max_total_tokens` xuống `1024`.
