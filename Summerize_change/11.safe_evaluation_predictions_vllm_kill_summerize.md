# 11. Safe Evaluation, Local Predictions, and Manual vLLM Kill Summary

## Safe Official Evaluation

- Đã sửa `source/evaluation/evaluate.py` để inference không crash khi thiếu official evaluator script.
- Khi `source/evaluation/official_evaluation/<dataset>/...` không tồn tại, code sẽ fallback sang `evaluate_by_dicts(...)`.
- Fallback metrics vẫn được ghi ra file official evaluation JSON.
- File JSON fallback có thêm:
  - `official_evaluation_skipped: true`
  - `official_evaluation_skip_reason: <reason>`
- Với MuSiQue hiện tại, repo thiếu:
  - `source/evaluation/official_evaluation/musique/evaluate_v1.0.py`
- Vì vậy `test_official_evaluation.json` hiện là internal evaluation fallback, không phải official MuSiQue evaluator thật.

## Manual vLLM Kill Script

- Đã thêm:

```text
script/kill_vllm_processes.py
```

- Script mặc định chỉ dry-run và liệt kê process liên quan vLLM.

```bash
python script/kill_vllm_processes.py
```

- Muốn kill thật:

```bash
python script/kill_vllm_processes.py --yes
```

- Muốn match cả parent process chạy `source.run.inference` hoặc `source.run.train`:

```bash
python script/kill_vllm_processes.py --include-rescore-runs --yes
```

- Code train/inference không được gắn auto-kill vLLM. Nếu thấy log `Killing local vLLM worker processes`, đó là lifecycle nội bộ của vLLM khi process lỗi hoặc shutdown, không phải logic kill thủ công do repo gọi.
