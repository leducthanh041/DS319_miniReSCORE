# 02. Python File Summary

## Phạm vi tóm tắt

File này chỉ tóm tắt các chỉnh sửa gần đây trong các file Python, bắt đầu từ đoạn sửa `dense.py` và các file `.py` được chỉnh sau đó.

Các file nằm trong phạm vi này:
- `source/module/retrieve/dense.py`
- `source/run/generate_passage_embeddings.py`
- `source/run/preprocess_raw_data.py`

## 1. `source/module/retrieve/dense.py`

### Vấn đề

Khi chạy bước `generate_passage_embeddings`, code đi vào nhánh:
- `passage_model_name_or_path != None`

Nhưng trong nhánh này code cũ:
- gán model passage nhầm vào `self.query_model`
- gán tokenizer passage nhầm vào `self.query_tokenizer`
- sau đó gọi `self.passage_model.cuda()` trước khi `self.passage_model` được khởi tạo

Hậu quả:
- lỗi runtime:
  - `AttributeError: 'DenseRetriever' object has no attribute 'passage_model'`

### Đã sửa

- tạo đúng:
  - `self.passage_model = AutoModel.from_pretrained(...)`
  - `self.passage_tokenizer = AutoTokenizer.from_pretrained(...)`
- sau đó mới gọi:
  - `self.passage_model = self.passage_model.cuda()`

### Ý nghĩa

Nhánh passage encoder giờ hoạt động đúng khi bước embedding truyền:
- `query_model_name_or_path=args.model_name_or_path`
- `passage_model_name_or_path=args.model_name_or_path`

## 2. `source/run/generate_passage_embeddings.py`

### 2.1. Sửa lỗi cấu hình embedding

Trước đó file này có một số điểm thiếu an toàn:
- chưa fail sớm khi input passages rỗng
- chưa bộc lộ rõ tiến độ trong log file

Đã sửa:
- báo lỗi rõ nếu không load được passage nào
- dùng đúng:
  - `--model_name_or_path`
  - `--no_fp16`
  - `--passage_maxlength`

### 2.2. Thêm progress tracking để theo dõi trong log

Đã thêm các mốc theo dõi:
- progress khi đọc file passages
- log tổng số passages đã load và shard đang xử lý
- progress khi convert passages thành `Document`
- progress khi chạy batch embedding bằng `tqdm`
- các `print(..., flush=True)` để log cập nhật ngay

Mục tiêu:
- giúp phân biệt giữa:
  - tiến trình thật sự đang chạy chậm
  - và tiến trình bị treo / crash

### 2.3. Sửa parser TSV để chịu được dữ liệu HotpotQA lớn

Sau khi thêm progress loader, bước đọc `hotpotqa.tsv` bị lỗi:
- `IndexError: list index out of range`

Nguyên nhân:
- một số dòng TSV không còn đúng giả định parser cũ kiểu:
  - `row[0], row[1], row[2]`
- dữ liệu text có thể chứa tab ẩn, làm lệch số cột

Đã sửa:
- bỏ cách parse cứng bằng `csv.reader([line], delimiter="\t")`
- đổi sang parser theo cấu trúc:
  - tách `id` bằng `split("\t", 1)`
  - tách `title` bằng `rsplit("\t", 1)`
- nếu vẫn có dòng malformed:
  - log warning
  - skip dòng đó thay vì crash toàn bộ job

### 2.4. Sửa gọi embedding passage

Đã dùng rõ ràng:
- `input_type='passage'`
- `verbose=True`

Mục tiêu:
- bật progress bar ở tầng embedding
- tránh việc log dừng im tại giai đoạn embedding

## 3. `source/run/preprocess_raw_data.py`

### 3.1. Sửa mismatch tên thư mục HotpotQA corpus

Code đã được chỉnh để chấp nhận cả:
- `data/raw_data/hotpotqa/wikipedia-paragraphs`
- `data/raw_data/hotpotqa/wikpedia-paragraphs`

Mục tiêu:
- tránh lỗi preprocess rỗng do khác tên thư mục raw corpus

### 3.2. Báo lỗi rõ nếu preprocess HotpotQA không sinh passage

Đã thêm kiểm tra:
- nếu sau preprocess số passage bằng `0`
- raise lỗi rõ ràng thay vì để bước embedding fail ở phía sau

### 3.3. Chuẩn hóa field trước khi ghi TSV

Đã thêm helper:
- `sanitize_tsv_field()`

Helper này thay:
- `\n` -> khoảng trắng
- `\t` -> khoảng trắng

Áp dụng cho các dataset đang ghi TSV:
- `hotpotqa`
- `iirc`
- `2wikimultihopqa`
- `musique`

### Ý nghĩa

Việc sanitize này giúp:
- `data/embed_ready_data/*.tsv` ổn định hơn
- tránh lỗi lệch cột khi parser TSV đọc lại ở bước `generate_passage_embeddings`

## 4. Chuỗi lỗi và cách các bản vá liên kết với nhau

Thứ tự các lỗi chính trong các file `.py` gần đây:

1. `dense.py`
   - passage model khởi tạo sai
   - gây `AttributeError`

2. `generate_passage_embeddings.py`
   - thiếu tracking nên khó biết job đang chạy hay treo
   - sau khi thêm tracking thì lộ ra lỗi parse TSV

3. `preprocess_raw_data.py`
   - TSV có thể chứa tab ẩn trong text/title
   - cần sanitize để các lần preprocess sau sinh file sạch hơn
