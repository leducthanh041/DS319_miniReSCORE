# 07. Resume Training Database Path Override Summary

## Context

Trong quá trình resume training từ checkpoint tốt nhất của `musique`, lỗi xảy ra:

```text
FileNotFoundError: Database directory not found: ./data/database/musique
```

Nguyên nhân là khi resume từ checkpoint `best_validation`, nếu truyền sai hoặc để config suy luận tự động từ model path, `PipelineConfig.database_path` có thể suy ra sai thư mục Retrieval DB.

Retrieval DB thực tế đang nằm tại:

```text
./data/database/contriever_msmarco/musique
```

Đường này là symlink sang local disk:

```text
/docker/data/thanhld/ReSCORE/data/database/contriever_msmarco/musique
```

## Files Changed

### `source/pipeline/config.py`

Đã thêm field mới:

```python
database_path_override: Optional[str] = None
```

Trong property `database_path`, nếu `database_path_override` được truyền vào thì dùng trực tiếp path này:

```python
if self.database_path_override:
    return self.database_path_override
```

Mục đích là tách rõ:

- `retrieval_query_model_name_or_path`: checkpoint/query encoder để tiếp tục train.
- `retrieval_passage_model_name_or_path`: passage encoder model, ví dụ `facebook/contriever-msmarco`.
- `database_path`: thư mục Retrieval DB chứa `docstore.db`, `index.faiss`, `faiss_id_to_docstore_id.pkl`.

### `source/run/train.py`

Đã thêm CLI flag:

```bash
--database_path
```

Flag này override trực tiếp thư mục Retrieval DB:

```bash
--database_path ./data/database/contriever_msmarco/musique
```

Trong `build_pipeline_config(args)`, nếu người dùng truyền `--database_path`, code sẽ gán:

```python
cfg_kwargs["database_path_override"] = args.database_path
```

## Correct Resume Command

Lệnh resume từ checkpoint tốt nhất của `musique` nên dùng:

```bash
CUDA_VISIBLE_DEVICES=1,2,3,4 python -m source.run.train \
  --running_name train_musique_resume_best \
  --dataset musique \
  --method rescore \
  --prompt_set 1 \
  --batch_size 20 \
  --retrieval_buffer_size 8 \
  --retrieval_count 4 \
  --max_num_thought 3 \
  --generation_max_batch_size 1 \
  --generation_max_total_tokens 2048 \
  --generation_max_memory_per_gpu 7GiB \
  --retrieval_query_model_name_or_path ./predictions/musique/train_musique___llama_3.1_8b_instruct___contriever_msmarco/multi_retrieval___train/prompt_set__1/retr_count__4/best_validation \
  --retrieval_passage_model_name_or_path facebook/contriever-msmarco \
  --database_path ./data/database/contriever_msmarco/musique \
  --retrieval_batch_size 16 \
  --retriever_device cuda:3 \
  --validation_freq 100 \
  --validation_batch_size 2 \
  --validation_max_batches 20 \
  --save_freq 500 \
  --early_stopping \
  --early_stopping_patience 5 \
  --early_stopping_min_delta 1e-4 \
  --num_workers 4
```

## Important Note

Không truyền Retrieval DB folder vào:

```bash
--retrieval_passage_model_name_or_path
```

Flag đó dùng cho passage encoder model, không phải FAISS/SQLite DB path.

Nếu muốn dùng DB nằm trên `/docker`, vẫn nên truyền path repo symlink:

```bash
--database_path ./data/database/contriever_msmarco/musique
```

vì `./data/database` hiện đã trỏ sang `/docker/data/thanhld/ReSCORE/data/database`.

## Scope

Thay đổi này chỉ hỗ trợ chọn đúng Retrieval DB khi resume/warm-start từ retriever checkpoint.

Nó chưa phải true resume đầy đủ vì code vẫn chưa lưu/khôi phục:

- optimizer state,
- scheduler state,
- epoch/step hiện tại,
- early stopping counter.

Đây là warm-start từ checkpoint retriever tốt nhất, không phải resume training bit-by-bit từ đúng batch bị lỗi.
