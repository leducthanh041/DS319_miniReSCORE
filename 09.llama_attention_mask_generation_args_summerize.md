# 09. Llama Attention Mask And Generation Args Summary

## Context

Sau khi dùng `eos_token` làm `pad_token` để tránh OOM tại `resize_token_embeddings()`, Transformers sinh warning:

```text
The attention mask is not set and cannot be inferred from input because pad token is same as eos token.
```

Ngoài ra còn có warning:

```text
do_sample is set to False, but temperature is set to 0.0
do_sample is set to False, but top_p is set to 0.9
```

Nhận định:

- Không nên quay lại cách cũ là thêm `"<PAD>"` rồi gọi `resize_token_embeddings()`, vì đây là nguyên nhân gây OOM khi load Llama.
- Cách đúng hơn là vẫn dùng EOS làm PAD, nhưng truyền `attention_mask` rõ ràng vào `model.generate()`.
- Với greedy decoding (`do_sample=False`), không nên truyền sampling args như `temperature`.

## Files Changed

### `source/module/generate/llama.py`

Đã sửa phần `_generate()` khi `use_vllm=False`.

Trước đó gọi:

```python
generated_outputs = self.model.generate(
    input_ids=model_inputs['input_ids'],
    **generation_args
)
```

Hiện tại gọi:

```python
generated_outputs = self.model.generate(
    input_ids=model_inputs['input_ids'],
    attention_mask=model_inputs.get('attention_mask'),
    **generation_args
)
```

Điều này loại warning attention mask và giúp generation ổn định hơn khi:

```python
pad_token_id == eos_token_id
```

Đã cleanup generation args:

```python
do_sample = self.cfg.temperature is not None and self.cfg.temperature > 0.
```

`generation_args` luôn có:

```python
"do_sample": do_sample
```

Nhưng chỉ truyền `temperature` nếu sampling thật sự bật:

```python
if do_sample:
    generation_args["temperature"] = self.cfg.temperature
```

Với config hiện tại `temperature=0.0`, `do_sample=False`, code sẽ không truyền `temperature`, tránh warning không cần thiết.

## Why Not Restore Original PAD Behavior

Không nên quay lại:

```python
self.tokenizer.add_special_tokens({'pad_token': "<PAD>"})
self.model.resize_token_embeddings(len(self.tokenizer))
```

Lý do:

- Thêm token mới làm thay đổi vocabulary size.
- `resize_token_embeddings()` phải khởi tạo embedding/lm_head mới.
- Với Llama 8B sharded trên GPU 11GB, bước resize này đã gây OOM trước khi training bắt đầu.
- Dùng EOS làm PAD là cách phổ biến cho causal LM inference/generation, miễn là truyền `attention_mask`.

## CUDA Memory Map Notes

Với lệnh:

```bash
CUDA_VISIBLE_DEVICES=1,2,3,4
--generation_max_memory_map 0:7GiB,1:7GiB,2:7GiB
--retriever_device cuda:3
```

mapping nội bộ là:

```text
cuda:0 -> physical GPU 1
cuda:1 -> physical GPU 2
cuda:2 -> physical GPU 3
cuda:3 -> physical GPU 4
```

Ý nghĩa:

- Generator Llama dùng `cuda:0`, `cuda:1`, `cuda:2`.
- Retriever dùng `cuda:3`.
- `generation_max_memory_map` không đảm bảo chia đều VRAM tuyệt đối.
- `cuda:0` vẫn có thể chiếm VRAM nhiều hơn vì thường là input device và nơi chứa một số module/tensor đầu tiên.
- Runtime tensors trong scoring/generation có thể làm VRAM `cuda:0` tăng thêm ngoài phần model placement.

Nếu `cuda:0` quá căng, có thể giảm riêng GPU 0:

```bash
--generation_max_memory_map 0:5GiB,1:7GiB,2:7GiB
```

Nếu vẫn thiếu, có thể thử CPU offload:

```bash
--generation_max_memory_map 0:5GiB,1:7GiB,2:7GiB,cpu:32GiB
```

nhưng CPU offload có thể làm train chậm hơn.
