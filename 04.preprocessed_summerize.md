# 04. Preprocessed Summary

## Mục tiêu của thay đổi

Tách rõ 2 nhánh dữ liệu trong repo:

- `processed_data/*.jsonl`
  - dùng cho `train.py` và `inference.py`
  - chứa question, contexts, answers, reasoning_steps
- `embed_ready_data/*.tsv` + `data/database/*`
  - dùng để build Retrieval DB
  - chứa corpus passage cho embedding và FAISS/docstore

Ý chính:

- `build.sh` không nên tạo `processed_data`
- `processed_data` cần được chuẩn bị trước hoặc bằng một script riêng

---

## Những gì đã thay đổi

### 1. `build.sh` quay về đúng vai trò

File:
- `script/download/build.sh`

Thay đổi:
- giữ `build.sh` chỉ cho pipeline build Retrieval DB:
  - `preprocess_raw_data`
  - `generate_passage_embeddings`
  - `build_index`
- bỏ việc gắn thêm bước tạo `processed_data` vào `build.sh`
- thêm log note để nói rõ:
  - script này chỉ build `data/embed_ready_data` và `data/database`
  - nếu thiếu `data/processed_data` thì phải chạy script riêng

Kết quả:
- logic của `build.sh` khớp lại với tư tưởng ban đầu của repo
- tránh nhầm giữa preprocessing cho corpus và preprocessing cho QA train/inference

---

### 2. Thêm script riêng cho `processed_data`

File:
- `script/download/multihop_processed_data.sh`

Thay đổi:
- tạo script riêng để chuẩn bị `data/processed_data` cho:
  - `hotpotqa`
  - `musique`
  - `2wikimultihopqa`
- script này:
  - ghi log ra `logs/preprocess/processed_data/<timestamp>/`
  - có resume/skip nếu output đã tồn tại
  - chia thành 2 bước:
    - `preprocess_processed_data`
    - `sample_processed_data`

Pipeline của script:
- gọi các script trong `preprocess/`
- sau đó gọi `preprocess/subsample_dataset_and_remap_paras.py`
  - để sinh:
    - `dev_subsampled.jsonl`
    - `test_subsampled.jsonl`

Kết quả:
- `processed_data` được chuẩn bị độc lập với build Retrieval DB
- dễ debug hơn vì có log riêng

---

### 3. Sửa `preprocess/process_hotpotqa.py` để dùng raw data local

File:
- `preprocess/process_hotpotqa.py`

Thay đổi:
- bỏ phụ thuộc vào `datasets.load_dataset(...)`
- chuyển sang đọc trực tiếp:
  - `data/raw_data/hotpotqa/hotpot_train_v1.1.json`
  - `data/raw_data/hotpotqa/hotpot_dev_distractor_v1.json`
- vẫn giữ output JSONL đúng schema cũ:
  - `question_id`
  - `question_text`
  - `answers_objects`
  - `contexts`
  - metadata như `level`, `type`

Kết quả:
- preprocess HotpotQA chạy được chỉ với raw data local
- không cần tải dataset từ ngoài trong bước này

---

### 4. Sửa `preprocess/lib.py` để giảm dependency lỗi không cần thiết

File:
- `preprocess/lib.py`

Thay đổi:
- chuyển một số import sang lazy import:
  - `_jsonnet`
  - `requests`
  - `rapidfuzz`

Lý do:
- các script preprocess cơ bản như `process_musique.py` và `process_2wikimultihopqa.py`
  không cần các dependency này ngay lúc import module

Kết quả:
- giảm khả năng fail sớm khi chỉ muốn chạy nhánh preprocess tạo `processed_data`

---

### 5. Cập nhật thông báo lỗi trong `train.py`

File:
- `source/run/train.py`

Thay đổi:
- nếu thiếu `data/processed_data/<dataset>/...`
  thì lỗi giờ gợi ý đúng hướng:
  - chạy `bash script/download/multihop_processed_data.sh`
  - hoặc chạy các script trong `preprocess/` + subsample script

Kết quả:
- dễ hiểu nguyên nhân hơn khi train fail vì thiếu processed QA data

---

### 6. Làm rõ ý nghĩa của `source/run/preprocess_raw_data.py`

File:
- `source/run/preprocess_raw_data.py`

Thay đổi:
- cập nhật docstring để nói rõ:
  - file này chỉ tạo `data/embed_ready_data/*.tsv`
  - không tạo `data/processed_data/*.jsonl`

Kết quả:
- tránh hiểu nhầm rằng `preprocess_raw_data` trong `build.sh` đã đủ cho train/inference

---

## Workflow mới nên dùng

### Bước 1. Tạo processed QA data

```bash
bash script/download/multihop_processed_data.sh
```

### Bước 2. Build Retrieval DB

```bash
bash script/download/build.sh
```

### Bước 3. Train

```bash
python -m source.run.train --running_name train_musique --dataset musique
```

## Kết luận

Sau thay đổi này:

- `processed_data` và `database` đã được tách rõ đúng vai trò
- `build.sh` quay về đúng mục đích ban đầu
- nhánh preprocess cho train/inference có script riêng, log riêng, và bám đúng ý tưởng trong thư mục `preprocess/`
