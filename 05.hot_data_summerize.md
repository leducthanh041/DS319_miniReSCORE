# 05. Hot Data / Docker Summary

## Mục tiêu

Xử lý nguyên nhân server bị nghẽn và tiến trình rơi vào `D-state` khi train/build:

- `NFS /mmlab_students` gần đầy, ghi liên tục rất chậm.
- `SQLite` đặt trên NFS dễ bị kẹt `file locking`.
- `logs`, `predictions`, `checkpoints`, `docstore.db` cần chuyển sang local SSD `/docker`.

## Những gì đã sửa

### 1. Bổ sung tối ưu SQLite

Đã sửa [source/module/index/docstore.py](/mmlab_students/storageStudents/nguyenvd/Thanhld/DS319/ReSCORE/source/module/index/docstore.py:42):

- thêm helper `_configure_connection()`
- luôn áp dụng:
  - `PRAGMA journal_mode=WAL`
  - `PRAGMA synchronous=NORMAL`
  - `PRAGMA temp_store=MEMORY`
  - `PRAGMA busy_timeout=60000`
- áp dụng cho cả:
  - lúc tạo DB mới
  - lúc `load()` DB có sẵn
- thêm `timeout=60` cho `sqlite3.connect(...)`
- đảm bảo `database_path` tồn tại trước khi mở DB

Mục tiêu:
- giảm áp lực I/O
- giảm rủi ro bị treo vì lock/ghi chậm trên filesystem

### 2. Thêm script migrate hot data sang `/docker`

Đã tạo [script/setup_local_hot_data.sh](/mmlab_students/storageStudents/nguyenvd/Thanhld/DS319/ReSCORE/script/setup_local_hot_data.sh:1)

Script này:
- giữ nguyên pipeline cũ
- chỉ di chuyển các thư mục ghi nặng:
  - `logs`
  - `predictions`
  - `data/database`
- sau đó tạo symlink ngược về repo

Mục tiêu:
- không phải sửa hàng loạt path trong code
- mọi chỗ đang ghi vào `./logs`, `./predictions`, `./data/database` sẽ tự động ghi sang local SSD

### 3. Train preflight: cảnh báo nếu vẫn ghi lên NFS

Đã sửa [source/run/train.py](/mmlab_students/storageStudents/nguyenvd/Thanhld/DS319/ReSCORE/source/run/train.py:331)

Train giờ sẽ in:
- `runtime_log_dir`
- `prediction_dir`
- `database_dir`
- `realpath` thật của từng thư mục
- cảnh báo nếu các path này chưa nằm trên `/docker`

Mục tiêu:
- tránh train nhầm khi symlink chưa được tạo
- nhìn log là biết hot data đang ghi ở đâu

### 4. Build preflight: cảnh báo nếu build DB vẫn ghi lên NFS

Đã sửa [script/download/build.sh](/mmlab_students/storageStudents/nguyenvd/Thanhld/DS319/ReSCORE/script/download/build.sh:38)

Build summary log giờ sẽ in:
- `build_log_dir`
- `database_root`
- `realpath` thật
- cảnh báo nếu vẫn chưa nằm trên `/docker`

Mục tiêu:
- tránh build `docstore.db`, `index.faiss`, log build trực tiếp trên NFS

### 5. Fail-fast rõ ràng khi `/docker` chưa có quyền

Đã cập nhật thêm trong [script/setup_local_hot_data.sh](/mmlab_students/storageStudents/nguyenvd/Thanhld/DS319/ReSCORE/script/setup_local_hot_data.sh:33):

- nếu user không có quyền traverse/write vào `/docker`
- script sẽ dừng sớm với hướng dẫn rõ cho admin
- không còn fail kiểu mơ hồ ở `mkdir: Permission denied`

## Kết quả kiểm tra thực tế hiện tại

Đã kiểm tra trực tiếp trên node hiện tại:

- user chạy là `thanhld`
- `/docker` hiện có quyền: `drwx--x--- nobody:nogroup`
- user `thanhld` chưa vào được `/docker/data/...`
- `test -w /docker/data/thanhld` trả `NOT_WRITABLE`

Kết luận:
- hiện tại repo **chưa migrate được**
- `logs`, `predictions`, `data/database` vẫn đang nằm ở `/mmlab_students/...`
- nguyên nhân không phải script sai
- nguyên nhân là **quyền filesystem của `/docker` chưa đúng**

## Việc cần làm tiếp theo

Admin cần cấp quyền filesystem đúng cho `/docker`, ví dụ:

```bash
sudo mkdir -p /docker/data/thanhld
sudo chown -R thanhld:thanhld /docker/data/thanhld
sudo chmod 711 /docker
sudo chmod 755 /docker/data
```

Sau đó tự kiểm tra lại:

```bash
ls -ld /docker /docker/data /docker/data/thanhld
test -w /docker/data/thanhld && echo WRITABLE || echo NOT_WRITABLE
```

Nếu ra `WRITABLE` thì chạy:

```bash
bash script/setup_local_hot_data.sh
realpath logs predictions data/database
```

Ba path trên phải trỏ về:

```text
/docker/data/thanhld/ReSCORE/...
```

Khi đó mới nên rerun:
- `bash script/download/build.sh`
- `python -m source.run.train ...`
